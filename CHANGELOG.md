## [0.4.0] - 2025-11-22

### Добавлено
- **3-уровневая архитектура проекта** для масштабируемости и переиспользования контента:
  - `1_universe/` - универсальный лор, применимый ко всем историям
  - `2_settings/` - переиспользуемые сеттинги миров
  - `3_stories/` - самодостаточные контейнеры историй
- Система версионирования проекта (файл `VERSION`)
- Детальный changelog для отслеживания всех изменений (этот файл)

### Изменено
- **Полная реструктуризация проекта** с использованием `git mv` для сохранения истории:
  - `lore/concepts/` → `1_universe/fundamental_concepts/` (7 файлов)
  - `lore/ludens_world/` → `1_universe/ludens_civilization/` (1 файл)
  - `Онтологи/` → `1_universe/ontology/` (8 файлов)
  - `entities/organizations/committee_designers.md` → `1_universe/institutions/committee_designers/overview.md`
  - `worlds/` → `2_settings/`
  - `entities/`, `story/`, `state/` → `3_stories/santa_cruz_incident/`
- Улучшена навигация между файлами благодаря чёткой иерархии

### Удалено
- Все избыточные папки старой структуры:
  - `lore/` (перенесено в 1_universe)
  - `worlds/` (перенесено в 2_settings)
  - `entities/` (перенесено в 3_stories/santa_cruz_incident)
  - `story/` (перенесено в 3_stories/santa_cruz_incident/narrative_structure)
  - `state/` (перенесено в 3_stories/santa_cruz_incident)
  - `locations/`, `settings/`, `spaces/` (пустые)
- Все пустые директории (автоматическая очистка)

### Технические детали
- **52 файла изменено**: 719 добавлений(+), 401 удалений(-)
- Все файлы перемещены с сохранением git-истории
- Структура готова для добавления новых историй без изменения базового лора

## Легенда

### Типы изменений
- **Добавлено** - новые возможности, файлы, функциональность
- **Изменено** - изменения существующей функциональности
- **Удалено** - удалённые возможности, файлы
- **Исправлено** - исправления багов
- **Безопасность** - изменения, связанные с безопасностью

### Формат версий
Проект использует [Semantic Versioning](https://semver.org/lang/ru/):
- **MAJOR.MINOR.PATCH** (например, 1.2.3)
- **MAJOR** (1.x.x) - несовместимые изменения API/структуры
- **MINOR** (x.2.x) - новая функциональность, обратно совместимая
- **PATCH** (x.x.3) - обратно совместимые исправления

Пока проект в стадии начальной разработки (0.x.x), структура может значительно меняться.
