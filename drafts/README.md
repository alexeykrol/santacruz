# /drafts - Рабочая версия (апрувленный контент)

Эта папка содержит **апрувленную рабочую версию** романа.

## ⚠️ Важно: Это НЕ черновики авторов

**Авторы работают в своих Git-ветках**, а не в этой папке.

**Workflow:**
1. Автор создает ветку (например, `author/alexey/episode005`)
2. Автор пишет эпизод в своей ветке
3. Автор создает **Pull Request**
4. После **апрува** PR сливается в `main` → попадает в `/drafts/`

**Drafts = канон, рабочая версия проекта**

---

## Структура

### [/master/](./master/) - Книжная версия
Единый текст, организованный как книга:
- Разбит по частям (Часть 1, Часть 2)
- Разбит по главам
- Разбит по эпизодам
- Единое повествование

### /episodeXXX/ - Эпизоды для рассылки
Каждая папка = один эпизод для почтовой рассылки:
- **Длина эпизода:** 10-12 тысяч символов
- **Содержание:** 1-3 сцены по необходимости
- **Формат:** Готов к публикации в рассылке

---

## Минимальная единица: Эпизод

**Эпизод** - это минимальная публикуемая единица для почтовой рассылки.

### Структура эпизода

```
/episode001/
├── README.md           # Описание эпизода (метаданные)
├── scene_001.md       # Сцена 1
├── scene_002.md       # Сцена 2 (опционально)
└── scene_003.md       # Сцена 3 (опционально)
```

### Метаданные эпизода

Каждый эпизод содержит:
- **Номер эпизода**
- **Название**
- **Часть и акт** (к какой части романа относится)
- **Длина** (количество символов)
- **Количество сцен** (1-3)
- **Краткое описание**
- **Статус** (черновик / готов к публикации / опубликован)

## Workflow: От ветки автора до публикации

```
1. Автор работает в ветке:
   author/alexey/episode005

2. Pull Request → Ревью → Апрув

3. Слияние в main → попадает в /drafts/episodeXXX/

4. Автоматическая сборка в /drafts/master/
   (эпизод интегрируется в книжную версию)

5. Публикация:
   - Эпизод → почтовая рассылка
   - Master → книжный формат
```

## Связь drafts и story

| Папка | Назначение | Формат |
|-------|------------|--------|
| **/drafts/** | Рабочая версия (апрувленная) | Эпизоды для рассылки + книжная версия |
| **/story/** | Архив опубликованных сцен | Каноничная структура (акты, сцены) |

**Возможная стратегия:**
- `/drafts/` - работа над текущими эпизодами
- После публикации эпизода → сцены архивируются в `/story/`
